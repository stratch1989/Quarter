## Критические баги (блокеры)

### 1. [CRASH] Повреждённый JSON в SharedPreferences крашит приложение при запуске
**Где:** `loadData()`, строки 1922-1934
**Проблема:** Парсинг JSON (`JSONObject(catNotesJson)`) не обёрнут в `try/catch`. Если SharedPreferences содержат невалидный JSON (например, после прерванной записи, ручного редактирования, или бага), `JSONObject()` бросит `JSONException` и приложение крашнется при запуске, войдя в бесконечный цикл краша (данные не очищаются).
**Ожидание:** graceful fallback -- пустой маппинг при невалидном JSON.
**Рекомендация:**
```kotlin
try {
    val jo = JSONObject(catNotesJson)
    jo.keys().forEach { k -> categoryNotesMap[k] = jo.getString(k) }
} catch (_: Exception) {
    categoryNotesMap.clear()
}
```

## Средние баги

### 2. [LOGIC] Автозаполнение не учитывает текст в noteField, который пользователь уже набирает
**Где:** строки 752-757, 850-855, 979-984
**Проблема:** Условие `activeNote.isNullOrEmpty()` проверяет переменную `activeNote`, а не содержимое `binding.noteField.text`. Переменная `activeNote` синхронизируется с noteField только при потере фокуса (строки 306-313) или нажатии IME_ACTION_DONE (строки 316-324). Сценарий:
1. Пользователь нажимает на noteField, начинает набирать текст (но не нажимает Done и не убирает фокус)
2. Затем нажимает на категорию (через dispatchTouchEvent фокус снимается, activeNote синхронизируется)

Фактически: `dispatchTouchEvent` (строка 1428) перехватывает тач вне noteField, вызывает `clearFocus()`, что триггерит `setOnFocusChangeListener` (строка 306), который синхронизирует `activeNote`. После этого клик по категории НЕ обрабатывается (строка 1436: `return true` поглощает событие). Пользователю нужен ВТОРОЙ тап.

**Вердикт:** Баг UX, но не потеря данных. При втором тапе `activeNote` уже синхронизирован, и автозаполнение корректно НЕ перезаписывает. Однако пользователь может быть запутан "поглощённым" первым тапом.

### 3. [LOGIC] Пустая заметка при наличии маппинга -- маппинг не удаляется, но и не обновляется
**Где:** строки 1323-1328
**Проблема:** Условие `activeCategory != null && !activeNote.isNullOrEmpty()` означает:
- Если пользователь осознанно стёр заметку (activeNote = null) и потратил, старый маппинг остаётся.
- Это может быть как фичей (сохранить для следующего раза), так и багом (пользователь хотел "забыть" заметку для этой категории).

**Вердикт:** Скорее дизайн-решение, но нет способа удалить маппинг кроме long-press удаления самой emoji. Нет возможности "очистить" привязанную заметку без удаления категории.

## Мелочи

### 4. [STYLE] Дублирование кода автозаполнения в 3 местах
**Где:** строки 752-757, 850-855, 979-984
**Проблема:** Одна и та же логика автозаполнения (выбор map, проверка savedNote != null && activeNote.isNullOrEmpty(), установка activeNote и noteField) повторяется в трёх местах: ввод через клавиатуру, клик по emoji в панели, выбор из пикера. При изменении логики нужно менять в 3 местах.
**Рекомендация:** Извлечь в приватный метод `applyAutoNote(emoji: String)`.

### 5. [MINOR] `saveCategoryNotesMap()` использует отдельный SharedPreferences.Editor
**Где:** строки 1819-1824 vs 1826-1862
**Проблема:** `saveCategoryNotesMap()` создаёт собственный `editor` и вызывает `apply()` независимо от `saveData()`. При `apply()` Android делает read-modify-write по конкретным ключам, так что конфликта нет. Однако это лишняя дисковая запись. Можно было бы сохранять маппинг в рамках `saveData()`.
**Степень:** Косметика, не влияет на корректность.

### 6. [MINOR] `JSONObject(categoryNotesMap as Map<*, *>)` -- unsafe cast
**Где:** строка 1821-1822
**Проблема:** Каст `as Map<*, *>` формально не нужен -- `MutableMap<String, String>` уже реализует `Map<*, *>`. Конструктор `JSONObject(Map)` принимает `Map<*, *>`. Работает корректно, но lint может предупредить об unchecked cast.

### Edge cases

| Кейс | Результат | Детали |
|------|-----------|--------|
| Emoji из нескольких code points (флаги, составные) | OK | `JSONObject` использует стандартные Java-строки, Unicode code points корректны как ключи |
| Длинная заметка (50+ символов) | OK | `JSONObject.put(String, String)` не имеет ограничений на длину значения |
| Спецсимволы: кавычки, \n, emoji в заметке | OK | `JSONObject.toString()` экранирует спецсимволы (`\"`, `\\n`), `getString()` декодирует обратно |
| Пустой JSON при первом запуске | OK | Строка 1924: `if (!catNotesJson.isNullOrEmpty())` -- при null/empty парсинг не запускается, маппинг остаётся пустым |
| Повреждённый JSON | **FAIL** | См. критический баг #1 -- нет try/catch |
| null в activeCategory | OK | Строка 1324: `if (activeCategory != null && ...)` |
| Миграция: старые пользователи без ключей | OK | `getString("CATEGORY_NOTES_MAP", null)` вернёт null, `isNullOrEmpty()` вернёт true, парсинг не запускается |
